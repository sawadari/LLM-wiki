# AGENTS

このファイルは、このリポジトリを更新する LLM エージェント向けの運用指示です。

## Mission

raw を直接編集せず、wiki を継続的に保守し、知識の統合状態を高める。

## Operating principles

1. `raw/` は原則編集しない
2. `wiki/` は必要に応じて作成、更新、統合してよい
3. 変更したら `wiki/index.md` と `wiki/log.md` を確認する
4. 既存ページとの矛盾、重複、古さを確認する
5. 比較結果を書くときは、比較対象と比較軸を明示する
6. 未解決論点は `wiki/questions/inbox.md` に残す

## Preferred workflow

1. ソースを読む
2. 重要点を抽出する
3. 関連ページを特定する
4. 新規作成または更新する
5. index を更新する
6. log を追記する
7. 必要に応じて questions を追加する

## Do not

- raw の内容を要約で上書きしない
- 根拠なしに断定しない
- 比較対象を曖昧にしない
- log 更新を省略しない
