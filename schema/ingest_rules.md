# Ingest Rules

## 目的

新しい source を追加したときに、wiki を一貫した形で更新する。

## 手順

1. source の種類を確認する
2. source の主張、事実、観点、未解決点を抽出する
3. 既存 wiki のどのページに影響するか確認する
4. 必要なら新規ページを作る
5. 既存ページの矛盾、重複、古さを確認する
6. index を更新する
7. log を追記する

## 最低限残すもの

- source の要点
- 既存理解との差分
- 強化された主張
- 揺らいだ主張
- 新たな問い

## 判断ルール

- 単なる言い換えなら既存ページへ統合する
- 新しい概念なら concept page を作る
- 新しい比較が発生したら comparison page 候補にする
- 結論を急がず、論点分離を優先する
